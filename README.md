# 🇧🇷 Olist End-to-End Data Pipeline
### *Modern Data Stack: BigQuery, dbt, Dagster, & Streamlit*

This project implements a professional-grade data engineering pipeline for **Olist**, the largest department store in Brazilian marketplaces. It transforms raw, fragmented e-commerce data into a **Star Schema** to provide actionable business insights through an automated, orchestrated, and documented workflow.

## 🚀 Key Features
* **Infrastructure:** Scalable cloud storage using **Google BigQuery**.
* **Transformation:** Modular SQL modeling using **dbt (data build tool)** for T (Transform) in ELT.
* **Orchestration:** End-to-end pipeline management and error handling via **Dagster**.
* **Data Quality:** Automated data testing and validation using **Great Expectations**.
* **Analytics:** Customer segmentation (RFM-inspired) and trend analysis.
* **UI/Visualization:** Interactive executive dashboard built with **Streamlit**.
* **Governance:** Fully searchable data dictionary and lineage generated by **dbt docs**.

---

## 🏗️ Architecture
The data follows a "Medallion-style" architecture:
1. **Bronze (Raw):** Olist CSV files loaded into BigQuery landing tables.
2. **Silver (Staging):** Cleaned and renamed views using dbt (`stg_` models).
3. **Gold (Warehouse):** Production-ready Star Schema with `fact_sales`, `dim_products`, and `dim_customers`.
4. **Reporting:** A flattened `view_executive_dashboard` optimized for BI tools.

## 📈 Dashboard Insights
The **Streamlit Executive Dashboard** provides real-time access to:
* **KPIs:** Total Revenue, Order Volume, and Average Order Value (AOV).
* **Geospatial Trends:** Sales performance across different Brazilian states.
* **Category Analysis:** Top 10 performing product categories by revenue.
* **Customer Segments:** Automatic classification into "VIP", "Loyal", and "Big Spender".

## 🚦 Getting Started

### 1. Clone & Setup
```bash
git clone [https://github.com/your-username/olist-data-pipeline.git](https://github.com/your-username/olist-data-pipeline.git)
cd olist-data-pipeline
pip install -r requirements.txt
```

### 2. Configure Credentials
Place your Google Cloud Service Account key in `credentials/gcp-key.json`. *(Note: Ensure this directory is listed in your `.gitignore` to prevent exposing your keys!)*

### 3. Run Orchestration (Dagster)
```bash
cd src/orchestration
dagster dev
```

### 4. Launch Dashboard
```bash
cd src
streamlit run app.py
```

## 📖 Documentation
